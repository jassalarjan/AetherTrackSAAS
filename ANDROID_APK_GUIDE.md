## Step 4 — Firebase Setup

### 4a. Create a Firebase Project
1. Go to https://console.firebase.google.com
2. Create project → **AetherTrack**
3. Add an Android app with package name `com.aethertrack.app`
4. Download `google-services.json`
5. Place it at `android/app/google-services.json`  
   ⚠️ Add to `.gitignore` — never commit this file

### 4b. Set environment variables for backend
```bash
# In backend/.env
FIREBASE_PROJECT_ID=your-project-id
FIREBASE_CLIENT_EMAIL=firebase-adminsdk-xxxxx@your-project.iam.gserviceaccount.com
FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"
```

### 4c. Enable Cloud Messaging
1. Firebase Console → Project Settings → Cloud Messaging
2. Copy Server Key → save as `FIREBASE_SERVER_KEY` in backend env
3. Register the backend `/api/notifications/register-device` route in `server.js`:

```js
// backend/server.js — add after other routes
const mobileRoutes = require('./routes/mobile');
app.use('/api', mobileRoutes);
```

## Step 6 — Integrate Mobile Hooks into App.jsx

```jsx
// frontend/src/App.jsx
import { useMobileCapabilities }   from './hooks/useMobileCapabilities';
import { usePushNotifications }    from './hooks/usePushNotifications';
import { registerSocketLifecycleHandlers } from './utils/mobileSocketManager';
import { initAutoUpdate }          from './utils/autoUpdate';
import { useEffect }               from 'react';
import { useAuth }                 from './context/AuthContext';

function AppInner() {
  const { user, token } = useAuth();
  const mobile = useMobileCapabilities();

  // Push notifications (only after login)
  usePushNotifications({ userId: user?._id, authToken: token });

  useEffect(() => {
    registerSocketLifecycleHandlers();
    initAutoUpdate();
  }, []);

  // Offline banner
  useEffect(() => {
    const showOffline = () => { /* show toast */ };
    const hideOffline = () => { /* hide toast */ };
    window.addEventListener('aether:offline', showOffline);
    window.addEventListener('aether:online',  hideOffline);
    return () => {
      window.removeEventListener('aether:offline', showOffline);
      window.removeEventListener('aether:online',  hideOffline);
    };
  }, []);

  return <RouterOutlet />;
}
```

## Step 7 — Replace localStorage with Secure Storage

In your AuthContext, replace token storage:

```js
// Before (insecure on native)
localStorage.setItem('token', token);
const token = localStorage.getItem('token');

// After
import { saveTokens, getAccessToken } from '../utils/secureTokenStorage';
await saveTokens({ accessToken, refreshToken });
const token = await getAccessToken();
```

## Step 9 — Add Launcher Icons & Splash Screen

### Icons (required sizes for Android adaptive icons)
Place PNG files in `android/app/src/main/res/`:
```
mipmap-mdpi/    ic_launcher.png         48×48
mipmap-hdpi/    ic_launcher.png         72×72
mipmap-xhdpi/   ic_launcher.png         96×96
mipmap-xxhdpi/  ic_launcher.png         144×144
mipmap-xxxhdpi/ ic_launcher.png         192×192
mipmap-xxxhdpi/ ic_launcher_round.png   192×192
```

### Adaptive Icon (Android 8+)
```
mipmap-anydpi-v26/
  ic_launcher.xml         ← references background + foreground layers
  ic_launcher_round.xml
drawable/
  ic_launcher_background.png   ← 1024×1024 background color fill
  ic_launcher_foreground.png   ← 1024×1024 icon on transparent bg
```

### Splash Screen
Place a `1280×1920` image at:
`android/app/src/main/res/drawable/splash.png`

Use `capacitor-assets` for automatic generation:
```bash
npx capacitor-assets generate --android
```

## Step 10 — Configure Deep Linking

### App Links (HTTPS) — requires domain verification
1. Create `frontend/public/.well-known/assetlinks.json`:
```json
[{
  "relation": ["delegate_permission/common.handle_all_urls"],
  "target": {
    "namespace": "android_app",
    "package_name": "com.aethertrack.app",
    "sha256_cert_fingerprints": ["YOUR_SHA256_FINGERPRINT"]
  }
}]
```
2. Get fingerprint: `keytool -list -v -keystore android/aethertrack.keystore -alias aethertrack`
3. Deploy to: `https://aethertrack.app/.well-known/assetlinks.json`

### Custom Scheme (works without domain ownership)
URLs like `aethertrack://dashboard` automatically open the app and navigate
via the `capacitorDeepLink` event wired in `MainActivity.java`.

## Step 11 — Build the APK

### Debug APK (development / testing)
```bash
npm run mobile:build          # builds React + syncs to Android
npm run apk:debug             # ./gradlew assembleDebug
npm run apk:install           # install to connected device/emulator
```

Debug APK location:  
`android/app/build/outputs/apk/debug/app-debug.apk`

### Release APK (production)
```bash
npm run mobile:build
npm run apk:release           # ./gradlew assembleRelease
npm run apk:sign              # node scripts/sign-apk.js
```

Signed APK location:  
`android/app/build/outputs/apk/release/app-release-signed.apk`

### AAB (for Google Play)
```bash
npm run apk:bundle            # ./gradlew bundleRelease
```
AAB location:  
`android/app/build/outputs/bundle/release/app-release.aab`

## Step 12 — CI/CD Pipeline (GitHub Actions example)

```yaml
# .github/workflows/android.yml
name: Build Android APK

on:
  push:
    tags: ['v*']

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-java@v4
        with: { distribution: 'temurin', java-version: '17' }

      - uses: actions/setup-node@v4
        with: { node-version: '20', cache: 'npm' }

      - name: Install root deps
        run: npm ci

      - name: Install frontend deps
        run: cd frontend && npm ci

      - name: Build React app
        run: cd frontend && npm run build
        env:
          VITE_API_URL:    ${{ secrets.VITE_API_URL }}
          VITE_WS_URL:     ${{ secrets.VITE_WS_URL }}
          VITE_UPDATE_URL: ${{ secrets.VITE_UPDATE_URL }}

      - name: Sync Capacitor
        run: npx cap sync android

      - name: Decode Keystore
        run: |
          echo "${{ secrets.KEYSTORE_BASE64 }}" | base64 -d > android/aethertrack.keystore

      - name: Build & Sign Release APK
        run: |
          cd android
          ./gradlew assembleRelease
        env:
          KEYSTORE_PATH:     android/aethertrack.keystore
          KEYSTORE_PASSWORD: ${{ secrets.KEYSTORE_PASSWORD }}
          KEY_ALIAS:         ${{ secrets.KEY_ALIAS }}
          KEY_PASSWORD:      ${{ secrets.KEY_PASSWORD }}

      - name: Sign APK
        run: node scripts/sign-apk.js
        env:
          ANDROID_HOME:      ${{ env.ANDROID_SDK_ROOT }}
          KEYSTORE_PATH:     android/aethertrack.keystore
          KEYSTORE_PASSWORD: ${{ secrets.KEYSTORE_PASSWORD }}
          KEY_ALIAS:         ${{ secrets.KEY_ALIAS }}
          KEY_PASSWORD:      ${{ secrets.KEY_PASSWORD }}

      - name: Upload APK artifact
        uses: actions/upload-artifact@v4
        with:
          name: AetherTrack-release
          path: android/app/build/outputs/apk/release/app-release-signed.apk
```

## OTA Updates (Frontend Deploy Without New APK)

When you push new React code:

```bash
cd frontend && npm run build  # produce dist/

# Zip the dist folder
zip -r aethertrack-bundle-1.0.1.zip dist/

# Upload to your CDN (e.g. S3, Cloudflare R2, etc.)
# Then update these backend env vars:
LATEST_BUNDLE_VERSION=1.0.1
LATEST_BUNDLE_URL=https://cdn.aethertrack.app/bundles/1.0.1.zip
LATEST_BUNDLE_CHECKSUM=sha256_of_the_zip
```

The app checks for updates on every foreground event and downloads automatically.
The new bundle activates on next app restart.

## Performance Checklist

| Item | Implementation |
|------|---------------|
| Code splitting | `manualChunks` in vite.config.js |
| Image CDN | Cloudinary (existing) |
| Lazy routes | `React.lazy()` + `<Suspense>` on all page imports |
| WebView preload | `android:hardwareAccelerated="true"` in AndroidManifest |

| Offline queue | Workbox BackgroundSync for mutations |

| Target modern Chrome | `target: ['es2020', 'chrome90']` in Vite |

## Troubleshooting

**WebView blank on first load**  
→ Ensure `npx cap sync android` was run after `npm run build`

**Socket.IO not connecting**  
→ Check `allowNavigation` in `capacitor.config.ts` includes your WSS URL  
→ Verify `androidScheme: 'https'` is set (required for secure cookies)

**FCM token not registering**  
→ Confirm `google-services.json` is in `android/app/`  
→ Check Firebase project has Cloud Messaging API enabled  
→ Verify `POST_NOTIFICATIONS` permission is granted on Android 13+

**Deep links not opening the app**  
→ For custom scheme: check `aethertrack://` intent filter in AndroidManifest  
→ For App Links: deploy `assetlinks.json` and verify with Google's validator

**Build fails with "Cannot resolve symbol 'R'"**  
→ Run Android Studio → Tools → Android → Sync Project with Gradle Files

**APK size too large (>50 MB)**  
→ Enable ProGuard + R8 in release build  
→ Use `bundleRelease` (AAB) instead of `assembleRelease` for Play Store  
→ Check `chunkSizeWarningLimit` and split charts into lazy-loaded routes
